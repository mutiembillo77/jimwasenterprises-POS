================================================================================
  HOW IT WORKS
================================================================================

1. USER LOGIN
   ├─ User authenticates with credentials
   ├─ User object created with role_code ('cashier', 'manager', 'admin')
   └─ Permissions loaded from database

2. PERMISSION INITIALIZATION
   ├─ Role permissions loaded on app start
   ├─ Cached in-memory for performance
   └─ Cache cleared on role changes

3. AUTHORIZATION CHECK
   ├─ Before accessing feature: hasPermission(userId, permissionName)
   ├─ Check returns boolean (true/false)
   ├─ Feature hidden or error shown if denied
   └─ Audit logged for all permission changes

4. MENU FILTERING
   ├─ Menu items filtered by user role
   ├─ getMenuItemsForRole(role) returns accessible items
   ├─ Sections shown/hidden based on role
   └─ Only available actions displayed to user

================================================================================
  PERMISSION FUNCTIONS
================================================================================

```javascript
// Check specific permission
await hasPermission(userId, 'sales.void')

// Check any permission (OR logic)
await hasAnyPermission(userId, ['sales.void', 'sales.refund'])

// Check all permissions (AND logic)
await hasAllPermissions(userId, ['sales.view', 'inventory.view'])

// Check domain-based permission
await canPerform(userId, 'sales', 'void')

// Get user permissions
const permissions = await getUserPermissions(userId)

// Check role permissions
await roleHasPermission('cashier', 'sales.void')

// Clear cache when role changes
clearPermissionCache(userId)
clearAllPermissionCache()
```
